C#概述(一)

根据我看书的情况大致写了一点C#的特点，后期会有专题详细讲解一些内容

程序结构

  • C#程序由一个或多个源文件组成。
  • 程序->类型->成员，可按命名空间进行组织。
  • 类与接口是类型的示例;字段、方法、属性、事件是成员的示例。
  • C#编译时将所有源文件一起处理。

类型与变量

  • 值类型：每个变量（除ref和out）都有自己的数据副本，对一个变量的操作不可能影响另一个变量。
  • 引用类型：变量存储对它们的数据的引用，可能两个引用类型变量存储相同数据的引用，即改变一个会改变另一个。
  • 使用类型声明创建新的类型，共有五种用户可定义类型。（详见C#教程，此处不赘述）

类和对象

类是最基础的C#类型。是一个数据结构，将状态和操作组合在一个单元中。使用new运算符创建它的实例。

  • 成员

    • 静态成员（属于类）与实例成员（属于对象）。

  • 可访问性

    可访问性 含义
    public 访问不受限制
    protected 访问仅限于此类或从此类派生的类
    internal 访问仅限于此程序
    protected internal 访问仅限于此程序或从此类派生的类
    private 访问仅限于此类

  • 基类

    • 类名或类型参数后面添加一个冒号和基类的名称来指定一个基类。不声明则基类是object。
    • 类继承基类成员，隐式地将其基类成员当作自己地成员（除个别构造函数）。

  • 方法

    静态方法通过类访问，实例方法通过累的实例来访问。

    方法签名（命名）同其它语言一样在文件中必须唯一。

    • 参数

      • 引用参数，与实参变量储存位置相同，修饰符为ref。
      • 输出参数，与引用参数类似，但实参初始值不重要，修饰符为out。
      • 参数数组，允许向方法传递可变数量的实参。例如WriteLine方法与Write方法：
        public class Console
{
       public static void Write(string fmt, params object[] args) {...}
       public static void WriteLine(string fmt, params object[] args) {...}
       ...
}

        其中若用户使用语句如下：
               Console.WriteLine("x={0} y={1} z={2}", x, y, z);

        则等价于：
               string s = "x={0} y={1} z={2}";
       object[] args = new object[3];
       args[0] = x;
       args[1] = y;
       args[2] = z;
       Console.WriteLine(s, args);

    • 方法体和局部变量

      • 方法体指定了在调用该方法时将执行的语句。
      • 局部变量就是仅在某方法中使用的变量。

    • 静态方法和实例方法

      • 静态方法不对特定实例进行操作，并且只能直接访问静态成员。
      • 实例方法对特定实例进行操作，并且能够访问静态成员和实例成员。
      • 静态方法中不能引用this
